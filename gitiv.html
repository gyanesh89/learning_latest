GIT Interview Cheat-Sheet (2024-2025)

30 questions + short answers + ready-to-run code snippets

1. Conceptual & Workflow

What is Git?

→ Distributed version-control system (DVCS) tracking content snapshots, not file diffs.

Difference between Git and GitHub?

→ Git = local tool; GitHub = hosted remote + collaboration layer.

Explain the three Git states.

→ Working directory → Staged (Index) → Repository (.git/objects).

Describe a typical Git flow (feature branch).

→ main → git checkout -b feature/x → commit → push → open PR → merge → delete branch.

How do you recover a deleted branch?

→ git reflog → find SHA → git checkout -b feature/x <SHA>.

2. Branching & Merging

6. Fast-forward vs. three-way merge?

→ FF moves pointer when history is linear; 3-way uses common ancestor when branches diverge.

Resolve merge conflict in CLI.

bash

git merge feature/x
# edit conflicted files
git add .
git commit

Rebase vs. merge?

→ Rebase rewrites history (linear); merge preserves original commits.

Interactive rebase to squash last 3 commits.

bash

git rebase -i HEAD~3
# change pick → squash for last 2

Force-push safely after rebase.

bash

git push --force-with-lease

3. Remote & Collaboration

11. Push a new local branch to origin.

bash

git push -u origin feature/x

Pull latest changes without merge commit.

bash

git pull --rebase

List all remotes.

bash

git remote -v

Add upstream repository.

bash

git remote add upstream https://github.com/org/repo.git

Fetch & rebase on upstream main.

bash

git fetch upstream
git rebase upstream/main

4. Stash, Reset & Clean

16. Save work-in-progress.

bash

git stash push -m "WIP login"

Apply most recent stash.

bash

git stash pop

Undo last commit but keep changes staged.

bash

git reset --soft HEAD~1

Discard all uncommitted changes.

bash

git reset --hard HEAD

Remove untracked files & dirs.

bash

git clean -fd

5. Hooks & Automation

21. Pre-commit hook to run black.

bash

# .git/hooks/pre-commit
#!/bin/sh
black --check .

GitHub Actions CI snippet.

yaml

name: lint
on: [push]
jobs:
  black:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: psf/black@stable

6. Advanced & Large Files

23. Store large files with Git LFS.

bash

git lfs install
git lfs track "*.pkl"
git add .gitattributes big.pkl
git commit -m "Add model"

Shallow clone to save bandwidth.

bash

git clone --depth 1 https://github.com/org/repo.git

Bisect to find bad commit.

bash

git bisect start
git bisect bad HEAD
git bisect good v1.0
# run test; git bisect good/bad until found

7. Cherry-pick & Tags

26. Apply a single commit from another branch.

bash

git cherry-pick <SHA>

Create & push annotated tag.

bash

git tag -a v2.0 -m "Release 2.0"
git push origin v2.0

Delete remote tag.

bash

git push --delete origin v2.0

8. Security & Signing

29. Sign commits with GPG.

bash

git config --global user.signingkey <KEY_ID>
git commit -S -m "feat: signed commit"

Verify signed tag.

bash

git tag -v v2.0
