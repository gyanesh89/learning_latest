MLOps and AIOps Interview Questions

MLOps Interview Questions

1. What is MLOps and how does it differ from DevOps?

MLOps extends DevOps by adding ML-specific stages: reproducible data pipelines, model versioning, continuous training, drift monitoring and rollback. While DevOps deals with deterministic code, MLOps must also handle stochastic data and model decay.

2. Key components of an end-to-end MLOps pipeline

  • Data & feature versioning (DVC / Delta Lake)
  • Experiment tracking (MLflow, Weights & Biases)
  • CI/CD for code + data + model (GitHub Actions + KubeFlow)
  • Feature store (Feast)
  • Model registry & canary deploy (SageMaker, Seldon)
  • Online/offline monitoring (Prometheus, Evidently AI)

3. How do you guarantee reproducibility?

  • Pin data (dvc add data.csv)
  • Pin code (git tag v1.2.3)
  • Pin env (Dockerfile + pip-tools)
  • Track with MLflow runs

import pandas as pd
from sklearn.model_selection import train_test_split

def load_split(path: str, test_size=0.2):
    df = pd.read_csv(path)
    return train_test_split(df.drop('y', 1), df['y'], test_size=test_size, random_state=42)

6. Code: minimal Flask REST API

Python

from flask import Flask, request, jsonify
import joblib

app = Flask(__name__)
model = joblib.load("model.pkl")

@app.route("/predict", methods=["POST"])
def predict():
    data = request.json["features"]
    pred = model.predict([data])[0]
    return jsonify(prediction=float(pred))

7. How do you detect data drift?

  • Offline: Population Stability Index (PSI) ≥ 0.2 → alert
  • Streaming: Kolmogorov-Smirnov test via Spark Streaming
  • Visual: Evidently AI dashboard

8. Automate retraining on new data

Airflow DAG:

schedule: "0 2 * * *"
tasks:
  1. check_drift >> 2. retrain >> 3. canary_deploy >> 4. rollback_if_drop>5%
Use KubernetesPodOperator with GPU nodes.

9. Model rollback vs roll-forward

Rollback = instant switch to last stable image (Kubernetes kubectl rollout undo). Roll-forward = push hot-fix version; preferred if fix is trivial.

10. Hyper-parameter tuning at scale

  • Bayesian optimisation (Optuna) on early-stop pruned trials
  • Run inside managed service (Vertex Vizier, SageMaker HPO) to parallelise 100s of GPUs

11. Secure ML in production

  • TLS + JWT for API auth
  • Docker rootless + readOnlyRootFilesystem
  • Hash checksum of model artifact on start-up
  • Scan SBOM with syft + grype nightly

12. Scenario: accuracy drop after deploy – checklist

  • Compare live vs training data distribution (Evidently)
  • Check feature pipeline logs for schema changes
  • Rollback via MLflow model registry if recall < SLA

AIOps Interview Questions

What is AIOps and why is it different from traditional monitoring?

AIOps applies AI/ML to IT-operations data to automate anomaly detection, root-cause analysis and predictive maintenance, going beyond static thresholds and dashboards.

List the three most common data inputs an AIOps platform ingests.

Metrics, logs and traces (the “three pillars” of observability).

Name two built-in correlation engines in IBM Cloud Pak for AIOps.

Scope-based grouping and temporal grouping.

Give an example of a scope-based grouping policy out-of-the-box.

Group alerts that share the same hostname within a 900-second rolling window.

How does probable-cause analysis work in AIOps?

An unsupervised algorithm scans events and topology to identify the most likely root-cause alert, then surfaces it as the parent incident.

Code: detect seasonality in alert counts

Python

import pandas as pd
from statsmodels.tsa.seasonal import seasonal_decompose

def detect_seasonality(series, period=24):
    """Return True if seasonal component amplitude > 0.3*std"""
    decomp = seasonal_decompose(series, model='additive', period=period)
    return decomp.seasonal.std() > 0.3 * series.std()

Code: basic anomaly detector on CPU %

Python

from sklearn.ensemble import IsolationForest

def cpu_anomaly(df):
    model = IsolationForest(contamination=0.02, random_state=42)
    df['anomaly'] = model.fit_predict(df[['cpu_percent']])
    return df[df['anomaly'] == -1]

Scenario / Troubleshooting

  • “Logs flood after a new release; how do you reduce noise?”
    Implement log-level throttling at the probe, drop debug/INFO events at source, and escalate only ERROR/FATAL via Netcool/Impact policy.
  • “Model accuracy for anomaly detection degrades after 30 days.”
    Schedule retraining during low-traffic periods and stagger model training runs at least one hour apart to avoid CPU spikes.
  • “Alerts double-count seasonal maintenance windows.”
    Enable alert seasonality detection; AIOps will auto-tag recurring events and suppress duplicate incidents.

Quick-Fire Tooling & Best-Practice Questions

  • Which storage topology is recommended for production AIOps?
    Dedicated SSD nodes (PCI passthrough) with scale-out design; avoid shared SAN/NAS.
  • What object-storage class should back-up AIOps backups?
    S3-compatible object storage located off-cluster.
  • How do you future-proof an AIOps deployment?
    Use platforms that auto-discover new devices and expose open APIs, ensuring seamless integration as new tools emerge.
