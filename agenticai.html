Understanding Agentic AI

Core Characteristics

Agentic AI represents a paradigm shift from traditional AI, embodying systems designed for autonomous operation and complex goal achievement. Unlike simple task execution, these systems demonstrate sophisticated reasoning, decision-making, and adaptive learning capabilities 270.

Operational Cycle

  1. 1
    Perception: Gathering data from diverse sources including sensors, databases, and APIs
  2. 2
    Reasoning: Analyzing data using LLMs and RAG to evaluate options and make decisions
  3. 3
    Action: Executing strategies by interacting with external tools via APIs
  4. 4
    Learning: Continuously improving performance through feedback and experience

Production-Level Python Code Examples

LangGraph Agent with FastAPI Template

A production-ready FastAPI template for building healthcare AI agents with LangGraph integration, featuring PostgreSQL persistence, Langfuse observability, and rate limiting 246. 

# app/core/agent.py - Healthcare Diagnostic Agent
from typing import TypedDict, Optional, Annotated
from langgraph.graph import StateGraph, END
from langchain_core.messages import HumanMessage
from langchain_community.tools.tavily_search import TavilySearchResults
from pydantic import BaseModel

class AgentState(TypedDict):
    patient_query: str
    medical_history: Optional[str]
    llm_response: Optional[str]
    next_step: Optional[str]
    final_answer: Optional[str]

# Define nodes for the workflow
def retrieve_medical_history(state: AgentState):
    # Simulate EHR integration
    return {"medical_history": "Patient history retrieved from EHR"}

def analyze_with_llm(state: AgentState, llm):
    messages = [
        HumanMessage(content=f"Patient query: {state['patient_query']}"),
        HumanMessage(content=f"Medical history: {state['medical_history']}")
    ]
    response = llm.invoke(messages)
    return {"llm_response": response.content}

def decide_next_step(state: AgentState):
    content = state["llm_response"].lower()
    if "schedule" in content:
        return {"next_step": "schedule_appointment"}
    elif "information" in content:
        return {"next_step": "provide_information"}
    return {"next_step": "escalate_to_human"}

# Build the workflow graph
def build_healthcare_agent(llm):
    workflow = StateGraph(AgentState)
    
    workflow.add_node("retrieve_history", retrieve_medical_history)
    workflow.add_node("analyze", lambda state: analyze_with_llm(state, llm))
    workflow.add_node("decide_step", decide_next_step)
    
    workflow.add_edge("retrieve_history", "analyze")
    workflow.add_edge("analyze", "decide_step")
    
    # Conditional edges based on next_step
    workflow.add_conditional_edges(
        "decide_step",
        lambda state: state["next_step"],
        {
            "schedule_appointment": "schedule_appointment",
            "provide_information": "provide_information",
            "escalate_to_human": END
        }
    )
    
    workflow.set_entry_point("retrieve_history")
    return workflow.compile()

# FastAPI endpoint
@app.post("/diagnose")
async def diagnose_patient(query: str):
    agent = build_healthcare_agent(llm)
    result = agent.invoke({"patient_query": query})
    return {"diagnosis": result["llm_response"], "next_step": result["next_step"]}

Databricks Mosaic AI Agent Framework

Implementing an investment assistant agent using Databricks Mosaic AI Framework with Unity Catalog integration and MLflow deployment 561. 

# quickstart_agent.py - Mosaic AI Agent
from databricks.agents import ChatAgent, ChatAgentResponse
from langchain_core.messages import HumanMessage
from langchain_community.chat_models import ChatDatabricks
import mlflow
import pandas as pd

class InvestmentAssistant(ChatAgent):
    def __init__(self):
        self.llm = ChatDatabricks(
            endpoint="databricks-claude-3-7-sonnet",
            temperature=0.1
        )
        self.tools = self._setup_tools()
    
    def _setup_tools(self):
        # Unity Catalog functions as tools
        return [
            {
                "name": "get_stock_summary",
                "description": "Retrieve stock summary information",
                "parameters": {"symbol": {"type": "string"}}
            },
            {
                "name": "get_historical_data",
                "description": "Get historical stock data",
                "parameters": {"symbol": {"type": "string"}, "days": {"type": "integer"}}
            }
        ]
    
    def predict(self, messages, **kwargs) -> ChatAgentResponse:
        latest_message = messages[-1].content
        prompt = f"""
        You are an expert investment assistant. Analyze the following request and 
        use appropriate tools to provide a detailed response.
        
        Request: {latest_message}
        """
        
        response = self.llm.invoke([
            HumanMessage(content=prompt)
        ])
        
        return ChatAgentResponse(
            message=response.content,
            metadata={"model": "databricks-claude-3-7-sonnet"}
        )

# Log and deploy the agent
with mlflow.start_run():
    agent = InvestmentAssistant()
    mlflow.pyfunc.log_model(
        "model",
        python_model=agent,
        extra_pip_requirements=["databricks-agents>=0.12.0"]
    )
    run_id = mlflow.active_run().info.run_id

# Deploy to Model Serving
from databricks.agents import deploy
deploy(run_id=run_id, model_name="investment_assistant")

# function_app.py
import azure.functions as func
from langgraph_func import LangGraphFunction

app = func.FunctionApp()

@app.function_name(name="HealthcareAgent")
@app.route(route="api/analyze", auth_level=func.AuthLevel.FUNCTION)
def healthcare_agent(req: func.HttpRequest) -> func.HttpResponse:
    agent = LangGraphFunction("healthcare.graph")
    result = agent.run(req.get_json())
    return func.HttpResponse(result)

Comparing Agentic AI Components

LangChain vs. LangGraph: Architectural Differences

LangChain

  • Simpler, sequential task execution
  • Linear control flows
  • Rapid prototyping focus
  • Single agent with tools

LangGraph

  • Complex, stateful workflows
  • Cyclical graphs with conditional branching
  • Multi-actor applications
  • Persistent memory management

Healthcare Case Study: Doctolib

Doctolib chose LangGraph over LangChain for their healthcare support system due to its ability to handle complex, stateful workflows, resulting in better cost management and reduced complexity 571.

MCP vs. A2A: Protocol Roles

Aspect MCP (Model Context Protocol) A2A (Agent-to-Agent Protocol)
Primary Communication Agent → Service/Tool/Data Source Agent ↔ Agent
Initiation Model Agent-initiated requests to services Bidirectional communication
Main Use Case Service interoperability, tool integration Multi-agent collaboration, task delegation
Discovery Mechanism Service capability metadata Agent cards describing skills
Typical Applications Calling APIs, querying databases Delegating tasks, coordinating workflows

Integration Synergy

MCP provides the "plumbing" for agents to connect to tools and data, while A2A provides the "language" for agents to collaborate. Together, they enable modular, scalable Agentic AI systems 425.
